Prototype vs Production Trust Boundary

Prototype Trust Model

Interceptor accepts direct function calls.
Session tokens are plain UUIDs. No request signing.
Tools simulated in-process. No network isolation.

Production Trust Model

Every interceptor call carries a signed capability token with a nonce.
Single-use nonce. TTL is seconds. Replay fails immediately.

Agent has zero credentials for any tool directly.
Tools accept only interceptor-signed requests.
VPC + mTLS + service mesh enforce network isolation.

State Store

Prototype: SQLite single node. Serializable on one machine.

Production: Cloud Spanner or CockroachDB.
Linearizable reads + serializable transactions across nodes.
Eventual consistency breaks FR-8 deterministically.

What This Does Not Solve

Physical coercion. OS-level compromise.
Cryptographic token forgery. Insider threat with DB admin access.

These are outside any application-layer system scope.
